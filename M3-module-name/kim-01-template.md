# КОНТРОЛЬНО-ИЗМЕРИТЕЛЬНЫЕ МАТЕРИАЛЫ (КИМ)

## Блок 3. Специфические задачи умного дома

**Дисциплина:** «Прикладной искусственный интеллект в системах умного дома»

---

## 1. Назначение КИМ

Контрольно-измерительные материалы предназначены для текущей и рубежной проверки достижения результатов обучения по всем компетенциям и индикаторам Блока 3: **ML-4.1**, **ML-2.2**, **O-3.1** и **AI S-1.1**.

---

## 2. Структура контрольной работы

| Часть | Содержание | Максимум |
|-------|------------|----------|
| А     | 20 тестовых заданий | 40 баллов |
| Б     | 2 аналитических задания | 20 баллов |
| В     | Практический кейс по аномалиям | 20 баллов |
| Г     | Кейс по энергоменеджменту или безопасности | 20 баллов |

---

## 3. Спецификация заданий по индикаторам

| Индикатор | Номера заданий | Доля |
|-----------|----------------|------|
| ML-4.1    | А1–А8, Б1, В   | 40 % |
| ML-2.2    | А9–А12, Б2     | 20 % |
| O-3.1     | А13–А15, Г1    | 20 % |
| AI S-1.1  | А16–А20, Г2    | 20 % |

### Часть А. Тестовые задания

**А1.** Основная цель Predictive Maintenance:

**А2.** Признак, наиболее характерный для Signature Analysis двигателя:

Б) цвет корпуса  
В) спектральные компоненты тока/вибрации  
Г) IP-адрес

**А3.** Isolation Forest обнаруживает аномалии за счет:

В) правил ассоциации  
Г) ручной разметки

**А4.** Значение -1 в sklearn IsolationForest означает:  
А) норму  
Б) пропуск  
В) аномалию  
Г) ошибку программы

**А5.** В Autoencoder аномальность обычно определяется по:  
А) ошибке реконструкции  
Б) имени класса  
В) размеру файла  
Г) числу эпох как таковому

**А6.** Порог Autoencoder выбирают по:  
А) случайному числу  
Б) распределению ошибки на нормальных/валидационных данных  
В) IP-адресу  
Г) числу столбцов

**А7.** Ложноположительный алерт — это:  
А) пропущенная авария  
Б) корректная тревога  
В) нормальное событие, признанное аномальным  
Г) удаление записи

**А8.** Для визуального анализа аномалий наиболее полезно:  
А) скрыть временную ось  
Б) отметить аномальные точки на временном ряду  
В) удалить подписи  
Г) заменить числа текстом

**А9.** К предварительной обработке относится:  
А) заполнение пропусков  
Б) изменение темы интерфейса  
В) переименование компьютера  
Г) отправка письма

**А10.** Масштабирование признаков необходимо, чтобы:  
А) признаки были сопоставимы по диапазону  
Б) увеличить число строк  
В) удалить модель  
Г) заменить датчики

**А11.** SOC батареи — это:  
А) скорость сети  
Б) состояние заряда  
В) стоимость тарифа  
Г) частота вибрации

**А12.** Пример производного признака для Demand Response:  
А) тариф × мощность  
Б) имя пользователя  
В) цвет графика  
Г) номер аудитории

**А13.** Конфликт в энергоменеджменте:  
А) стоимость энергии против комфорта  
Б) шрифт против размера окна  
В) имя против пароля  
Г) дата против времени

**А14.** Отклонение от оптимального режима означает:  
А) достижение всех целей  
Б) несоответствие фактических параметров целевым ограничениям  
В) отсутствие данных всегда  
Г) только поломку датчика

**А15.** При низком SOC и высоком тарифе сначала следует:  
А) оценить ограничения комфорта и ресурса батареи  
Б) всегда разрядить батарею до нуля  
В) отключить все датчики  
Г) удалить историю

**А16.** При Federated Learning исходные данные:  
А) передаются всем участникам  
Б) остаются локально  
В) публикуются в открытом доступе  
Г) не используются

**А17.** Differential Privacy используется для:  
А) увеличения размера модели  
Б) ограничения раскрытия информации об отдельных записях  
В) повышения напряжения  
Г) замены датчиков

**А18.** Data Poisoning — это:  
А) отравление/искажение данных обучения  
Б) отказ батареи  
В) сжатие модели  
Г) визуализация

**А19.** STRIDE применяется для:  
А) моделирования категорий угроз  
Б) расчета SOC  
В) кластеризации  
Г) построения спектра

**А20.** Риск упрощенно оценивают как:  
А) вероятность × последствия  
Б) тариф + температура  
В) число эпох × размер файла  
Г) SOC / IP
