▎Требования к добавлению бенчмарков

▎1. Описание задачи

• Формулировка: Четкое определение проблемы (классификация, регрессия, аномалия, прогнозирование).
• Контекст умного дома: Связь с реальным сценарием (управление климатом, безопасность, энергоэффективность).
• Входные данные: Тип сенсоров, частота дискретизации, размер датасета.
• Целевой выход: Формат предсказания (класс, вероятность, числовое значение, действие).

Пример:
 Задача: Прогнозирование потребления электроэнергии на 1 час вперед на основе исторических данных и метеорологических параметров. Входные данные: временной ряд мощности (15-минутная дискретизация), температура, влажность. Выход: точечный прогноз мощности в кВт.


---

▎2. Метрики оценки

• Основные метрики: MAE, RMSE, MAPE для регрессии; Accuracy, Precision, Recall, F1 для классификации.
• Метрики качества данных: Полнота (%), пропуски, выбросы.
• Метрики Edge AI: Размер модели (МБ), время инференса (мс), потребление памяти (МБ).
• Метрики безопасности: Время отклика, надежность в условиях помех.

Таблица метрик:


Метрика: RMSE
Формула / определение: √(1 / n∑ᵢ₌₁ⁿ(yᵢ - ŷᵢ)²)
Целевое значение: < 5% от диапазона
Приоритет: Высокий

Метрика: Latency (Edge)
Формула / определение: Время инференса на целевом устройстве
Целевое значение: < 100 мс
Приоритет: Высокий

Метрика: Model Size
Формула / определение: Размер .tflite или .onnx
Целевое значение: < 5 МБ
Приоритет: Средний

Метрика: F1-score
Формула / определение: (2 ⋅ Precision ⋅ Recall) / (Precision + Recall)
Целевое значение: > 0.85
Приоритет: Высокий

▎3. Протокол эксперимента

• Разбиение данных: Train/Val/Test (например, 60/20/20 или временная валидация).
• Кросс-валидация: k-fold (k=5) для малых датасетов.

> ChatGPT 5 | Gemini 3 | Nano Banana:
• Гиперпараметры: Фиксированные значения для воспроизводимости.
• Окружение: Python 3.9+, TensorFlow 2.10+, scikit-learn 1.0+, PyTorch 1.10+.
• Повторяемость: Фиксированные seed'ы (random_state=42).

Пример протокола:

1. Загрузить датасет из источника
2. Нормализация: StandardScaler (fit на train, применить на val/test)
3. Обучение на train (100 эпох, batch_size=32, lr=0.001)
4. Валидация на val (ранняя остановка при плато)
5. Финальная оценка на test (без утечки данных)
6. Сохранить модель и метрики в JSON

▎4. Базовые решения (бейзлайны)

Бейзлайн: Dummy Classifier
Описание: Предсказание наиболее частого класса
Реализация: `sklearn.dummy.DummyClassifier(strategy='most_frequent')`
Ожидаемая метрика: Accuracy ≈ 50%

Бейзлайн: Linear Regression
Описание: Линейная модель для регрессии
Реализация: `sklearn.linear_model.LinearRegression()`
Ожидаемая метрика: RMSE ≈ 15%

Бейзлайн: Decision Tree
Описание: Дерево решений (max_depth=5)
Реализация: `sklearn.tree.DecisionTreeClassifier(max_depth=5)`
Ожидаемая метрика: F1 ≈ 0.70

Бейзлайн: Random Walk
Описание: Для временных рядов: предсказание = последнее значение
Реализация: `ypred = ytest.shift(1)`
Ожидаемая метрика: MAPE ≈ 20%

Бейзлайн: MLP (1 слой)
Описание: Простая нейросеть: 64 нейрона, ReLU, Dropout(0.2)
Реализация: TensorFlow Keras
Ожидаемая метрика: F1 ≈ 0.75

Бейзлайн: TinyML Reference
Описание: Предобученная модель TensorFlow Lite для Edge
Реализация: `tflite_convert`
Ожидаемая метрика: Latency < 50 мс

▎6. Правила сравнения решений

• Справедливое сравнение: Одинаковые train/val/test, seed, версии библиотек.
• Статистическая значимость: Для малых датасетов — доверительные интервалы (95%).
• Компромисс точность-сложность: Учитывать размер модели и скорость инференса.
• Контекст умного дома: Приоритет надежности и задержке над абсолютной точностью.
