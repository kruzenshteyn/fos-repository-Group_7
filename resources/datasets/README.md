Название: UCI Energy Efficiency Data Set
Аннотация: Предсказание энергопотребления зданий на основе параметров конструкции и условий окружающей среды. 768 образцов, 8 признаков (площадь, высота стен, ориентация и т.д.), 2 целевые переменные (отопление, охлаждение). Используется для обучения регрессионных моделей управления климатом в умном доме.
Связанные КИМ: ТФ-MLE-1, ТФ-MLE-2, ТФ-MLE-3, ТФ-DE-4
Доступ: [UCI ML Repository](https://archive.ics.uci.edu/ml/datasets/energy+efficiency)
Лицензия / условия: CC BY 4.0
Дата проверки: 2024-01-15

Название: Appliances Energy Prediction Data Set
Аннотация: Временные ряды энергопотребления бытовых приборов с интервалом 10 минут. 19,735 образцов, 27 признаков (датчики температуры, влажности, освещённости), целевая переменная — мощность приборов. Демонстрирует работу с потоковыми IoT-данными и временными зависимостями.
Связанные КИМ: ТФ-DE-1, ТФ-DE-2, ТФ-MLE-3, ТФ-AIA-2
Доступ: [UCI ML Repository](https://archive.ics.uci.edu/ml/datasets/appliances+energy+prediction)
Лицензия / условия: CC BY 4.0
Дата проверки: 2024-01-15

Название: MNIST (модифицированный для Edge AI)
Аннотация: Распознавание рукописных цифр (28×28 px, 70,000 образцов). Используется для демонстрации развёртывания нейросетей на Edge-устройствах (Raspberry Pi, Arduino) с квантизацией и оптимизацией модели.
Связанные КИМ: ТФ-MLE-2, ТФ-MLE-3, ТФ-MLE-5, ТФ-AIA-3
Доступ: [TensorFlow Datasets](https://www.tensorflow.org/datasets/catalog/mnist) или [Yann LeCun](http://yann.lecun.com/exdb/mnist/)
Лицензия / условия: CC0 (Public Domain)
Дата проверки: 2024-01-15

Название: Home Assistant Community Datasets
Аннотация: Реальные данные с датчиков умных домов: температура, влажность, движение, освещённость, потребление электроэнергии. Различные временные масштабы и частоты дискретизации. Позволяет работать с неструктурированными и асинхронными потоками данных.
Связанные КИМ: ТФ-DE-1, ТФ-DE-3, ТФ-AIA-2, ТФ-AIA-4
Доступ: [Home Assistant Community](https://github.com/home-assistant/core) + примеры
Лицензия / условия: Apache 2.0
Дата проверки: 2024-01-15

Название: CIFAR-10 (для компьютерного зрения в умном доме)
Аннотация: Классификация изображений 32×32 px в 10 категориях (животные, транспорт и т.д.). 60,000 образцов. Применяется для обучения моделей распознавания объектов на камерах видеонаблюдения умного дома с ограниченными ресурсами.
Связанные КИМ: ТФ-MLE-2, ТФ-MLE-3, ТФ-AIA-3
Доступ: [CIFAR-10](https://www.cs.toronto.edu/~kriz/cifar.html)
Лицензия / условия: CC0 (Public Domain)
Дата проверки: 2024-01-15

Название: Occupancy Detection Data Set
Аннотация: Определение присутствия людей в помещении по датчикам (CO₂, температура, влажность, свет). 20,560 образцов, 5 признаков, бинарная классификация. Прямое применение для автоматизации освещения и климата.
Связанные КИМ: ТФ-MLE-1, ТФ-MLE-2, ТФ-MLE-4, ТФ-DE-4
Доступ: [UCI ML Repository](https://archive.ics.uci.edu/ml/datasets/occupancy+detection+)
Лицензия / условия: CC BY 4.0
Дата проверки: 2024-01-15

Название: Synthetic IoT Sensor Data (генерируемый)
Аннотация: Синтетические временные ряды с контролируемыми аномалиями, пропусками и шумом для обучения обнаружению неисправностей датчиков и отказоустойчивости. Позволяет симулировать сценарии отказа оборудования.
Связанные КИМ: ТФ-DE-2, ТФ-DE-3, ТФ-MLE-4, ТФ-AIA-5
Доступ: [Kaggle](https://www.kaggle.com) или собственная генерация
Лицензия / условия: MIT или CC BY 4.0
Дата проверки: 2024-01-15

Название: KITTI Dataset (для робототехники)
Аннотация: Данные с датчиков автономного автомобиля (камеры, LiDAR, GPS). Применяется для обучения моделей навигации и восприятия домашних роботов. 15 GB, 7,481 сцен.
Связанные КИМ: ТФ-MLE-2, ТФ-AIA-4, ТФ-AIA-5
Доступ: [KITTI Vision Benchmark Suite](http://www.cvlibs.net/datasets/kitti/)
Лицензия / условия: CC BY-NC-SA 3.0
Дата проверки: 2024-01-15


▎Рекомендации по использованию

▎Состав признаков и целевые переменные

• Числовые: температура, влажность, освещённость, мощность, CO₂
• Категориальные: тип помещения, режим работы, статус датчика
• Временные: временные метки, час суток, день недели

▎Объём и разбиение

• Обучение: 70% (случайная выборка или временная последовательность)
• Валидация: 15% (для настройки гиперпараметров)
• Тест: 15% (для финальной оценки и демонстрации)

▎Ограничения и смещения

• ⚠️ Дисбаланс классов: в задачах обнаружения аномалий нормальные состояния преобладают
• ⚠️ Сезонность: энергопотребление зависит от времени года
• ⚠️ Отсутствующие значения: реальные IoT-данные содержат пропуски
• ⚠️ Смещение распределения: модель, обученная на одном доме, может плохо работать на другом

▎Рекомендуемые техники обработки

• Нормализация / стандартизация признаков
• Обработка пропусков (интерполяция, forward-fill)
• Удаление выбросов (IQR, z-score)
• Кроссвалидация по временным окнам (для временных рядов)
