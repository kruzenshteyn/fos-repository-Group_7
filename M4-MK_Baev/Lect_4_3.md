1. Введение: экосистема умного дома и задачи интеграции (10 минут)
1.1. Проблема фрагментации
Современный умный дом — это экосистема, состоящая из устройств десятков производителей, работающих на разных протоколах и платформах. Пользователь сталкивается с проблемами:

Разрозненность управления — каждое устройство требует своего приложения

Отсутствие совместимости — устройства разных брендов не взаимодействуют

Сложность автоматизации — создание сквозных сценариев требует интеграции разнородных систем

Масштабируемость — добавление нового устройства часто требует перестройки всей системы

1.2. Роль платформ интеграции
Платформы типа Home Assistant и openHAB решают эти проблемы, предоставляя:

Единую точку управления всеми устройствами

Поддержку множества протоколов и стандартов

Механизмы автоматизации и создания сценариев

Возможность расширения через пользовательские интеграции

Инфраструктуру для внедрения ML-моделей — от сбора данных до исполнения решений

1.3. Связь с компетенцией LC‑5.2
Компетенция LC‑5.2 требует осуществлять выбор инструментов и инженерных практик по управлению данными с учётом:

2. Протоколы умного дома: Matter, Zigbee, Z-Wave, MQTT (25 минут)
2.1. Обзор протоколов
На рынке представлено множество протоколов, каждый со своими особенностями. Рассмотрим четыре ключевых.

2.2. Zigbee
Zigbee — один из наиболее зрелых и распространённых протоколов.

Характеристика	Описание
Частотный диапазон	2.4 ГГц
Топология	Ячеистая сеть (mesh)
Максимальное число узлов	До 65000
Энергопотребление	Низкое
Применение	Датчики, освещение, замки, розетки
Преимущества:

Широкая экосистема устройств (Philips Hue, IKEA, Honeywell)

Нативная поддержка в Amazon Echo, Samsung SmartThings

Надёжная mesh-сеть

Недостатки:

Разные профили устройств могут быть несовместимы

Требуется координатор (шлюз)

2.3. Z‑Wave
Z‑Wave — низкопотребляющий mesh-протокол, работающий в суб-гигагерцевом диапазоне.

Характеристика	Описание
Частотный диапазон	800‑900 МГц
Топология	Ячеистая сеть
Дальность	До 100 м (на открытом пространстве)
Совместимость	Более 300 производителей
Преимущества:

Меньше помех от Wi-Fi (другой диапазон)

Стандартизированный стек протоколов — устройства совместимы

Хорошая дальность

Недостатки:

Меньшая пропускная способность

Региональные различия частот

2.4. Matter
Matter — новый универсальный стандарт, разработанный Connectivity Standards Alliance (Amazon, Apple, Google, Samsung, Silicon Labs).

Характеристика	Описание
Транспорт	Wi‑Fi, Ethernet, Thread
Применение	Умные колонки, дисплеи, камеры, термостаты, замки, датчики
Совместимость	Межвендорная, межплатформенная
Ключевая идея Matter — создать универсальный стандарт, при котором устройство любого производителя работает с любой платформой.

Архитектура:

Прикладной уровень поверх IP (Thread или Wi‑Fi)

Устройства могут подключаться напрямую к экосистемам (Alexa, Google Home) без отдельного хаба

Поддержка локального управления — снижение задержки и повышение надёжности

2.5. MQTT
MQTT (Message Queuing Telemetry Transport) — лёгкий протокол обмена сообщениями по модели «издатель-подписчик».

Характеристика	Описание
Модель	Издатель-подписчик (Pub/Sub)
Транспорт	TCP/IP
Качество обслуживания	3 уровня (at most once, at least once, exactly once)
Применение	Телеметрия, IoT, интеграция систем
Почему MQTT важен для умного дома:

Лёгкость — минимальный оверхед, подходит для ограниченных устройств

Гибкость — может использоваться как «клей» между разными системами

Асинхронность — устройства обмениваются сообщениями без прямого соединения

Широко поддерживается — Home Assistant, openHAB, Node‑RED

from homeassistant.core import HomeAssistant
from homeassistant.config_entries import ConfigEntry
from homeassistant.const import Platform

DOMAIN = "my_integration"

PLATFORMS = [Platform.SENSOR, Platform.SWITCH]

async def async_setup_entry(hass: HomeAssistant, entry: ConfigEntry) -> bool:
    """Настройка интеграции из config entry."""
    hass.data.setdefault(DOMAIN, {})[entry.entry_id] = {}
    await hass.config_entries.async_forward_entry_setups(entry, PLATFORMS)
    return True

async def async_unload_entry(hass: HomeAssistant, entry: ConfigEntry) -> bool:
    """Выгрузка интеграции."""
    unload_ok = await hass.config_entries.async_unload_platforms(entry, PLATFORMS)
    if unload_ok:
        hass.data[DOMAIN].pop(entry.entry_id)
    return unload_ok

8.1. Ключевые выводы
Умный дом — это распределённая система с множеством протоколов и устройств

Matter — будущий стандарт межвендорной совместимости

MQTT — универсальная шина для интеграции разнородных систем

Home Assistant и openHAB — платформы, предоставляющие инфраструктуру для ML

Custom Components — ключевой механизм расширения функциональности
